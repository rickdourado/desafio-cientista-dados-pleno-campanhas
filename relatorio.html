Ranking e Algoritmo de Escolha

Ponto 3 · Ranking de Sistemas

Wilson Lower Bound — Ranking Estatisticamente Defensável

Em vez de usar a taxa simples (entregues / total), a solução usa o limite inferior do IC de Wilson a 95%. Esse método penaliza sistemas com poucos disparos, corrigindo o viés de volume.

WLB = (p̂ + z²/2n − z√[p̂(1−p̂)/n + z²/4n²]) / (1 + z²/n)
onde z = 1.96 (IC 95%) · p̂ = taxa observada · n = total de disparos
from scipy import stats

def wilson_lower_bound(positives: int, total: int, confidence: float = 0.95) -> float:
    """Limite inferior do IC de Wilson — mais robusto para volumes diferentes."""
    if total == 0:
        return 0.0
    z = stats.norm.ppf(1 - (1 - confidence) / 2)   # z ≈ 1.96
    phat = positives / total
    denominator = 1 + z**2 / total
    center = phat + z**2 / (2 * total)
    margin = z * np.sqrt(phat * (1 - phat) / total + z**2 / (4 * total**2))
    return (center - margin) / denominator
Fig. 3 — Ranking de sistemas usando Wilson Lower Bound (conservador). Sistemas com alto volume e boa taxa ficam no topo; sistemas com poucas amostras são penalizados.

🥈 2º Sistema B — 800 em 1.000 80% ≈ 77% Promovido
🧠

Por que Wilson LB é matematicamente superior? Com amostras pequenas, a taxa simples é altamente instável. Wilson LB incorpora a incerteza estatística — quem tem mais volume prova sua confiabilidade. Isso elimina o viés de seleção identificado na Parte 1.

def calcular_score_telefone(row, sistema_scores, hoje=pd.Timestamp.today()):
    # 1. Qualidade da fonte (Wilson LB normalizado)
    score_sistema = sistema_scores.get(row['id_sistema'], 0.0)

    # 2. Frescor — decai exponencialmente (meia-vida ~1 ano)
    dias = (hoje - pd.to_datetime(row['registro_data_atualizacao'])).days
    score_frescor = np.exp(-max(0, dias) / 365)

    # 3. DDD preferencial (Rio = 21)
    score_ddd = 1.0 if str(row['telefone_ddd']) == '21' else 0.7

    return 0.50 * score_sistema + 0.40 * score_frescor + 0.10 * score_ddd

Cálculo de Tamanho de Amostra
from scipy.stats import norm

def calcular_tamanho_amostra(taxa_base, delta, alpha=0.05, poder=0.80):
    """n mínimo por grupo para detectar melhoria de 'delta' pontos percentuais."""
    z_alpha = norm.ppf(1 - alpha)          # unicaudal → z ≈ 1.645
    z_beta  = norm.ppf(poder)               # poder 80% → z ≈ 0.842
    p_bar   = taxa_base + delta / 2
    n = (z_alpha + z_beta)**2 * 2 * p_bar * (1 - p_bar) / delta**2
    return int(np.ceil(n))

# Parâmetros: detectar +3 p.p. com α=5% e poder=80%
n_por_grupo = calcular_tamanho_amostra(taxa_base=0.30, delta=0.03)

Análise Pós-Experimento
from scipy.stats import proportions_ztest

def analisar_ab_test(entregues_A, total_A, entregues_B, total_B, alpha=0.05):
    taxa_A = entregues_A / total_A
    taxa_B = entregues_B / total_B
    lift   = (taxa_B - taxa_A) / taxa_A

    counts = np.array([entregues_B, entregues_A])
    nobs   = np.array([total_B, total_A])
    z_stat, p_valor = proportions_ztest(counts, nobs, alternative='larger')

    if p_valor < alpha:
        print(f"✅ SIGNIFICATIVO — rejeitar H₀. Lift: {lift:+.1%}. Implantar.")
    else:
        print(f"❌ NÃO significativo — coletar mais dados ou revisar.")
📏

Parâmetros do experimento: α = 5% · poder = 80% · delta mínimo detectável = +3 p.p. A duração estimada depende do volume diário real de disparos — calculada automaticamente pelo notebook.
